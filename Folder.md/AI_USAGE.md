# AI_USAGE.md — Использование AI в разработке

## Инструменты
- **AI-ассистент:** opencode (big-pickle модель)
- **Режим:** интерактивный CLI с доступом к файловой системе

## Как использовался AI

### 1. Генерация кода (80%)
AI использовался для генерации:
- Начальной структуры Next.js проекта
- Типов данных (types.ts)
- Схемы валидации (form-schema.ts)
- Mock-данных (mock-data.ts)
- Компонентов (Header, Footer, BookingForm, BookingTable, LanguageSwitcher)
- Страниц (page.tsx для всех роутов)
- Файлов переводов (en.json, ru.json)
- Конфигурации next-intl (routing, navigation, request, middleware)
- Статической HTML-версии (Index.html, style.css, script.js)
- Документации (TASK.md, PROMPT_PLAN.md, AI_USAGE.md, README.md)

### 2. Рефакторинг и доработка (15%)
- Разделение server/client components
- Вынос схемы валидации из JSX
- Вынос mock-данных из компонентов
- Исправление путей импортов
- Добавление недостающих полей в форму

### 3. Объяснение концепций (5%)
AI помогал объяснить:
- Принципы работы next-intl
- Разницу между server и client components
- Структуру App Router

## Что было доработано вручную

1. **Валидация формы** — добавлен `superRefine` для проверки количества пассажиров
2. **Переводы** — скорректированы некоторые формулировки для естественности
3. **Стили** — настроены кастомные цвета (gold, gray shades) под бренд
4. **Ссылки в Header/Footer** — проверены и исправлены пути

## Ограничения AI
- AI не настраивает и не запускает dev-сервер
- AI не тестирует приложение в браузере
- AI использует только те библиотеки, которые уже установлены в проекте
- Дизайн-решения принимались на основе текстового описания, без визуального макета
